# Contributing

This repository is currently documentation-first.

- Keep product, Ayurvedic domain, quiz, data, editorial, AI, design, and engineering decisions in their corresponding directories.
- Do not present a draft as expert-approved; retain document status and TODO markers until review is complete.
- Keep user-facing copy in `content/` and specifications in `docs/`.
- Version structured data changes and record consequential decisions in `decisions/`.
- Keep wellness guidance within `docs/ayurveda/safety-boundaries.md`.
