# Glossary Data Model

> Status: Provisional demo schema implemented.

Glossary entries are authored in `content/glossary/glossary.json`. Each requires a stable kebab-case ID, display term, short definition, aliases, a valid related article ID, content status, and publication status. Only published entries appear in Learn and Guided help.

The demo schema intentionally omits Sanskrit/transliteration detail, long definitions, named approval records, revision numbers, and per-term safety notes. Add those fields only with editorial and expert ownership. Current entries are provisional and visibly labeled through their destination content.
